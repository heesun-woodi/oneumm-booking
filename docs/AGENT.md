# 🏠 온음 공간 예약 자동화 AGENT

## 목표
온음 공간(놀터, 방음실) **예약 프로세스 자동화**

## 🎯 Agent Pattern
**Prompt Chaining** (프롬프트 체이닝)
- **고정된 단계**로 예약 프로세스 분해
- 각 단계가 명확하고 예측 가능
- 순차적으로 진행, 각 단계마다 검증

### 예약 흐름 (5단계)
```
입력 검증 → 예약 가능 확인 → 예약 생성 → 확인 메시지 → 캘린더 동기화
```

### 왜 이 패턴?
- 예약 프로세스는 **명확하게 정의된 단계**
- 각 단계의 입출력이 고정됨
- 중간에 검증 게이트가 필요 (예약 가능 여부 등)

## AGENT 역할

### Step 1: 입력 검증
**입력 받기**:
- 공간 유형 (놀터 / 방음실)
- 날짜 및 시간
- 예약자 정보 (이름, 연락처)
- 사용 목적 (선택)

**검증**:
- 날짜 형식 유효성
- 시간 범위 (운영 시간 내)
- 필수 정보 누락 확인

**출력**: ✅ 유효한 입력 → Step 2 / ❌ 오류 → 사용자에게 재요청

---

### Step 2: 예약 가능 확인
**체크 사항**:
- 해당 시간대 다른 예약 존재 여부
- 공간 점검/폐쇄 일정 확인
- 연속 예약 제한 규칙 (최대 2시간 등)
- 동시 예약 제한 (1인당 1개 등)

**데이터 소스**:
- `~/Documents/buzz-workspace/projects/oneumm-booking/calendar.json`
- `~/Documents/buzz-workspace/projects/oneumm-booking/rules.md`

**출력**: ✅ 예약 가능 → Step 3 / ❌ 불가 → 대체 시간 제안

---

### Step 3: 예약 생성
**작업**:
- 예약 데이터 생성 (고유 ID, 타임스탬프)
- 캘린더에 기록
- 데이터베이스 업데이트

**예약 데이터 구조**:
```json
{
  "id": "UUID",
  "space": "놀터|방음실",
  "date": "YYYY-MM-DD",
  "time_start": "HH:MM",
  "time_end": "HH:MM",
  "user": "이름",
  "contact": "연락처",
  "purpose": "사용 목적",
  "created_at": "timestamp",
  "status": "confirmed"
}
```

**출력**: 예약 확정 정보 → Step 4

---

### Step 4: 확인 메시지 생성
**메시지 포함 내용**:
- 예약 번호
- 공간 정보
- 날짜 및 시간
- 주의사항 (사용 규칙, 준비물 등)
- 취소/변경 방법

**메시지 톤**:
- 친절하고 명확함
- 필수 정보 강조
- 커뮤니티 감성

**출력**: 확인 메시지 → Step 5

---

### Step 5: 캘린더 동기화
**작업**:
- Google Calendar 또는 온음 공용 캘린더에 이벤트 추가
- 리마인더 설정 (1일 전, 1시간 전)
- 예약자에게 캘린더 초대 발송 (선택)

**출력**: ✅ 예약 완료!

---

## 취소/변경 흐름 (별도)

### 취소 프로세스
1. 예약 번호로 예약 조회
2. 취소 가능 여부 확인 (최소 2시간 전 등)
3. 예약 상태 변경 (`cancelled`)
4. 캘린더에서 제거
5. 취소 확인 메시지

### 변경 프로세스
1. 기존 예약 취소
2. 새 예약 생성 (동일한 흐름)

## 예약 규칙

### 공통 규칙
- 예약 단위: 1시간
- 최소 예약: 1시간 전
- 최대 예약: 2주 후까지
- 취소/변경: 최소 2시간 전

### 놀터 규칙
- 운영 시간: 09:00 - 21:00
- 연속 예약: 최대 2시간
- 1인당 주간 예약: 최대 3회

### 방음실 규칙
- 운영 시간: 09:00 - 22:00
- 연속 예약: 최대 3시간
- 1인당 주간 예약: 최대 5회
- 소음 주의 (22시 이후 사용 금지)
